# termproject

repo นี้เป็น workshop ก่อนลงแลปจริง ได้ทดลองคล้ายๆ ntier V2 บทเดียวกัน

สิ่งที่ได้เรียนรู้ใน Week นี้ คือ

Soft Ware ควรจะมี3ส่วนหลัก คือ 1. front 2.back 3.Database ก่อนจะออกแบบ soft Ware ควรจะเลือก Architectural ก่อน เช่น 1.เน้น performance 2.เน้น แข็งแรงทนทานการล่ม 3.Security เน้นความปลอดภัยของข้อมูล

Architectural หลักคือ 1.monolithic คือ เขียนโค้ดแบบรวมทุกอย่างไว้เป็นก้อนเดียวกันทั้ง front,back,database 2.Layered คือ จะเป็นการจัดระเบียบขึ้นจากรวมกันเป็นก่อน monolithic แต่ยังไม่ได้แยกกันมาก 3.client-server คือทำการประมวลผลแยกส่วนกันและจะส่งสิ่งที่ประมวลผลไปหากัน แบ่งได้2ลักษณะดังนี้ -state race คือ การทำงานโดยไม่เชื่อมต่อกับ backend -state full คือ การทำงานแบบเชื่อมต่อกับ backend ไปด้วย 4.N-Tier คือ จะแยกส่วนจาก Layered กันอย่างชัดเจนอยู่คนละที่

ความแตกต่าง VM และ Docker คือ Docker จะไม่มี OS แต่ต้องหาไปโหลด image ก่อน จากนั้นสั่งรันให้เป็น Contriner จากการทดลอง health check ยิ่งมี app เยอะจำยิ่งโหลดนานขึ้นใช้เวลานานขึ้น
